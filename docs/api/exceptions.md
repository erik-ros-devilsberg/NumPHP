# Exceptions

NumPHP defines four exception classes, all in the global namespace. They are registered in `MINIT` and form a small hierarchy rooted at `\NDArrayException`:

```
\RuntimeException
└── \NDArrayException
    ├── \ShapeException
    ├── \DTypeException
    └── \IndexException
```

The base class extends PHP core's `\RuntimeException`. Catching `\RuntimeException` catches every exception NumPHP throws. Catching `\NDArrayException` is more specific — it catches every NumPHP-thrown exception while ignoring unrelated runtime errors.

The exceptions live in the global namespace (not `numphp\…`) to follow the PHP core convention for `\RuntimeException`-derived exceptions thrown from extensions (e.g. `\ValueError`, `\TypeError`).

---

## \NDArrayException

Base class — and the catch-all for situations that don't fit the three subclasses.

**Thrown by:**

- `Linalg::inv` / `Linalg::solve` when the matrix is singular.
- `Linalg::eig` when an eigenvalue has non-zero imaginary part (decision 15).
- `NDArray::offsetUnset` (always — element removal is unsupported).
- `NDArray::concatenate` / `stack` if more than 64 inputs are given.
- `NDArray::bufferView` if the source is non-C-contiguous.
- `NDArray::clip` if both `min` and `max` are `null`.
- `NDArray::arange` if `step == 0`.
- File I/O failures (`save` / `load` / `toCsv` / `fromCsv`).
- Sort with all-NaN slice (`nanargmin`/`nanargmax`).

---

## \ShapeException

Shape or dimensionality mismatch.

**Thrown by:** any operation whose shape preconditions are violated. Examples:

- Broadcasting two arrays whose shapes are not broadcast-compatible (`add`/`subtract`/`multiply`/`divide` and friends).
- `matmul` / `dot` / `inner` / `outer` with the wrong rank or mismatched inner dimensions.
- `Linalg::*` with non-square or non-2-D inputs.
- `reshape` with a target whose product doesn't equal `size()`.
- `transpose` with an `$axes` that isn't a valid permutation.
- `concatenate` / `stack` with mismatched non-axis dimensions.
- `fromArray` on ragged input.
- `squeeze` on a named axis whose size is not 1.
- `expandDims` if the result would exceed 16 dimensions.

---

## \DTypeException

Unsupported or incompatible dtype.

**Thrown by:**

- `zeros` / `ones` / `full` / `eye` / `arange` / `fromArray` when `$dtype` is not one of `"float32"`, `"float64"`, `"int32"`, `"int64"`.
- `full` and `fromArray` when the input value cannot be represented in the requested dtype (e.g. `NaN` cast to `int32`).
- Element-wise ops where the operand is not a number or NDArray.

---

## \IndexException

Out-of-bounds index or invalid slice.

**Thrown by:**

- `offsetGet` / `offsetSet` / `offsetExists` (after negative-index normalisation).
- `slice` with `start > stop` or otherwise invalid bounds.

---

## Other PHP core exceptions

NumPHP also surfaces the following PHP-core exceptions where appropriate:

- `\DivisionByZeroError` — integer division by zero (`NDArray::divide` and the `/` operator on int dtypes). Note: float division by zero produces IEEE 754 results (`inf`, `nan`) and does not throw — see [decision 7](../system.md).
- `\TypeError` — when arguments fail PHP's type-hint check. Generated by the engine before NumPHP code runs.
